# How'd we get here?

A big part of the point of this project is lowering the impact of the "bus factor."

> The "bus factor" is a risk management metric indicating the minimum number of team members who, if suddenly unavailable (say, hit by a bus), would cause a project to stall due to a lack of key knowledge or skills.

A low bus factor (like 1) signifies high vulnerability. 

A significant goal of building-in-the-open is to record as much as possible about how we got here -- to mitigate the vulnerability of projects with a bus factor of 1.

If handoffs, design decisions, agentic building plans and so forth are all committed, it becomes trivial to walk back through the history to see how the current state came to be.

The `.handoffs` directory is used by coding agents to coordinate work across sessions. Once a handoff document has served its purpose, it can be moved to the `.bus-factor` directory as a way of preserving the timeline.
